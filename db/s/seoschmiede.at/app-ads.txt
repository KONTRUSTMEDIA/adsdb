  function escapeHtml(s){ return (s||'').replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;').replace(/'/g,'&#39;'); }
  function mdLite(text){
    const parts = String(text||'').split(/```/);
    let html = '';
    for (let i=0;i<parts.length;i++){
      if (i%2===1){ html += `<pre><code>${escapeHtml(parts[i])}</code></pre>`; continue; }
      const lines = parts[i].split(/\r?\n/);
      let inUL=false, para='';
      const closePara=()=>{ if (para.trim()) { html += `<p>${para.trim()}</p>`; para=''; } };
      const closeUL=()=>{ if (inUL){ html += `</ul>`; inUL=false; } };
      for (const raw of lines){
        const line = raw || '';
        if (/^\s*#{2,4}\s+/.test(line)){ closeUL(); closePara(); const level=(line.match(/^\s*(#{2,4})\s+/)||[])[1].length; html += `<h${level}>${escapeHtml(line.replace(/^\s*#{2,4}\s+/, ''))}</h${level}>`; }

        else { closeUL(); para += (para ? ' ' : '') + escapeHtml(line.trim()); }

					<h2 class="elementor-heading-title elementor-size-default">Was sind LLMs?</h2>				</div>
				<div class="elementor-element elementor-element-504606dc e-transform e-transform elementor-widget__width-initial elementor-widget-mobile__width-initial elementor-widget-divider--separator-type-pattern elementor-widget-divider--view-line elementor-widget elementor-widget-divider" data-id="504606dc" data-element_type="widget" data-e-type="widget" data-settings="{&quot;_transform_rotateZ_effect&quot;:{&quot;unit&quot;:&quot;px&quot;,&quot;size&quot;:1,&quot;sizes&quot;:[]},&quot;_transform_rotateZ_effect_hover&quot;:{&quot;unit&quot;:&quot;px&quot;,&quot;size&quot;:2,&quot;sizes&quot;:[]},&quot;_transform_rotateZ_effect_tablet&quot;:{&quot;unit&quot;:&quot;deg&quot;,&quot;size&quot;:&quot;&quot;,&quot;sizes&quot;:[]},&quot;_transform_rotateZ_effect_mobile&quot;:{&quot;unit&quot;:&quot;deg&quot;,&quot;size&quot;:&quot;&quot;,&quot;sizes&quot;:[]},&quot;_transform_rotateZ_effect_hover_tablet&quot;:{&quot;unit&quot;:&quot;deg&quot;,&quot;size&quot;:&quot;&quot;,&quot;sizes&quot;:[]},&quot;_transform_rotateZ_effect_hover_mobile&quot;:{&quot;unit&quot;:&quot;deg&quot;,&quot;size&quot;:&quot;&quot;,&quot;sizes&quot;:[]}}" data-widget_type="divider.default">

						</div>
				<div class="elementor-element elementor-element-2f4263ae elementor-widget elementor-widget-text-editor" data-id="2f4263ae" data-element_type="widget" data-e-type="widget" data-widget_type="text-editor.default">
									<p>Große Sprachmodelle, oft LLMs genannt, sind Programme, die aus sehr viel Text gelernt haben. Sie zerlegen Sprache in kleine Einheiten, sogenannte Tokens, erkennen darin Muster und Wahrscheinlichkeiten und können daraus Antworten bilden, die für Menschen gut lesbar sind. Im Kern arbeiten sie mit Transformer Modellen. Diese Modelle können Abhängigkeiten in einem Text weit über einzelne Sätze hinaus berücksichtigen und dadurch Zusammenhänge erkennen, die einfache Verfahren übersehen würden.</p><p>Beim Lernen durchlaufen LLMs meist zwei Phasen. Zuerst kommt das allgemeine Vortraining. Dabei liest das Modell große Mengen öffentlicher Texte, zum Beispiel Bücher, Webseiten und Foren. Das Ziel ist schlicht, die nächste sinnvolle Einheit in einer Folge vorherzusagen. In der zweiten Phase wird das Verhalten verfeinert. Dafür nutzt man kleinere Datenmengen mit Beispielen in hoher Qualität, oft von Menschen bewertet. Eine verbreitete Methode hört auf den Namen Reinforcement Learning from Human Feedback. Menschen vergleichen mehrere Modellantworten, das System lernt daraus, welche Variante hilfreicher, korrekter und höflicher wirkt.</p><p>Was LLMs gut können, zeigt sich im Alltag. Sie beantworten Fragen, erklären Fachbegriffe, fassen lange Dokumente zu Kurztexten zusammen und entwerfen E Mails oder Blogartikel. Sie helfen beim Programmieren, indem sie Funktionen skizzieren, Fehlermeldungen einordnen oder Code besprechen. Sie unterstützen bei Übersetzungen, bei der Suche nach Formulierungen und bei der Anpassung von Ton und Stil an eine Zielgruppe. Viele Modelle behalten dabei den Kontext eines laufenden Gesprächs. Wer eine Rückfrage stellt oder auf einen früheren Punkt verweist, bekommt eine Antwort, die sich darauf bezieht. Genau diese Gesprächsfähigkeit macht LLMs für Kundenservice, Wissensmanagement und Lernangebote interessant.</p><p>Damit Ergebnisse verlässlicher werden, lassen sich LLMs mit eigenen Quellen koppeln. Ein verbreiteter Ansatz heißt Retrieval Augmented Generation. Dabei sucht ein separates System zuerst in internen Dokumenten, Datenbanken oder Webseiten nach passenden Abschnitten. Diese Abschnitte werden dem Modell als zusätzlicher Kontext mitgegeben. Die Antwort stützt sich dann stärker auf überprüfbare Inhalte. Für Unternehmen ist das wichtig, weil so Richtlinien, Produktkataloge, Preislisten oder Support Anleitungen direkt in die Antwort einfließen können, ohne dass man das gesamte Modell neu trainieren muss.</p><p>Gute Anwendungsszenarien finden sich in vielen Bereichen. Im Support können Assistenten Tickets vorqualifizieren, bekannte Lösungen vorschlagen und Übergaben an Menschen sauber vorbereiten. Im Marketing entstehen Briefings, Texte für Landingpages, Anzeigenentwürfe und Varianten für A B Tests. In der Redaktion helfen LLMs bei Themenrecherche, bei der Struktur von Artikeln und bei der Zusammenfassung von Quellen. In der Software Entwicklung beschleunigen sie Code Reviews, Tests und Dokumentation. In der Analyse unterstützen sie bei der Beschreibung von Diagrammen, beim Entwurf von Abfragen und bei Notizen zu Ergebnissen. In der Lehre erstellen sie Übungsfragen, Beispielrechnungen und Erklärtexte in unterschiedlicher Tiefe.</p><p>Wie man Qualität sichert, ist ein zentraler Punkt. Erstens braucht es klare Ziele. Soll die Antwort kurz sein oder ausführlich. Soll sie neutral klingen oder werblich. Zweitens helfen feste Prüfregeln. Einfache Checks fragen, ob Quellen genannt sind, ob Zahlen im Text zu den Zahlen in den Quellen passen, ob Namen korrekt geschrieben sind. Drittens lohnt es sich, wichtige Themen mit sogenannten System Vorgaben zu steuern. Darin steht zum Beispiel, welche Inhalte tabu sind, wie mit sensiblen Daten umzugehen ist und wie mit Unsicherheit umzugehen ist. Viertens hilft Auswertung mit Metriken. Für Aufgaben mit klarer Lösung kann man Trefferquote, Genauigkeit oder F1 messen. Für offene Fragen nutzt man Vergleiche durch mehrere Menschen, die Kriterien wie Verständlichkeit, Korrektheit und Nützlichkeit bewerten.</p><p>Grenzen und Risiken sollte man offen ansprechen. LLMs können plausible, aber falsche Aussagen machen. Dieser Effekt wird oft Halluzination genannt. Er fällt stärker ins Gewicht, wenn Fragen sehr speziell sind oder wenn keine passenden Quellen vorliegen. Dazu kommen Verzerrungen aus Trainingsdaten. Manche Formulierungen sind in den Daten häufiger als andere und werden deshalb bevorzugt, auch wenn sie nicht die beste Wahl sind. Rechtliche Themen spielen ebenfalls eine Rolle. Unternehmen sollten klären, welche Daten in das System eingespeist werden dürfen, wie lange Daten gespeichert werden und wer Zugriff hat. Für sensible Inhalte empfiehlt sich strenge Protokollierung, eine klare Trennung zwischen internen und externen Umgebungen und ein Freigabeprozess durch Menschen.</p><p>Für den Einsatz in Organisationen hat sich ein wiederkehrendes Vorgehen bewährt. Zu Beginn steht eine kleine Machbarkeitsprüfung mit echten Beispielen aus dem Arbeitsalltag. Danach folgt eine Phase mit enger Rückkopplung an die Nutzer. In dieser Zeit verbessert man Prompts, ergänzt fehlende Quellen, legt Stil Leitfäden fest und passt die Schnittstellen an bestehende Werkzeuge an. Erst wenn Qualität und Nutzen stabil sind, skaliert man auf weitere Teams und Prozesse. In jeder Stufe gehören Messwerte dazu, zum Beispiel Bearbeitungszeit pro Anfrage, Zufriedenheit der Nutzer, Anteil korrekt gelöster Fälle und Aufwand für Nacharbeit.</p><p>Auch technische Entscheidungen wirken sich stark auf Ergebnis und Kosten aus. Ein großes Modell liefert oft die besten Antworten, ist aber teurer und langsamer. Kleinere Varianten sind günstiger und genügen für viele Aufgaben, wenn man sie mit guten Prompts und passenden Quellen verbindet. Caching kann Kosten senken, indem wiederkehrende Antworten gespeichert werden. Moderation vor der Anfrage filtert heikle Eingaben, Moderation nach der Antwort entdeckt problematische Ausgaben. Für sensible Umgebungen kommt eine Bereitstellung in einer isolierten Infrastruktur in Frage. Für öffentliche Inhalte reichen oft Dienste aus der Cloud mit strikter Konfiguration.</p><p>Ein Wort zum Schreiben mit LLMs. Texte wirken natürlicher, wenn man konkrete Beispiele und Zahlen nennt, kurze und längere Sätze abwechselt, Füllwörter vermeidet und direkte Verben nutzt. Statt Allgemeinplätzen helfen knappe Belege und klare Empfehlungen. Wenn ein Text unsicher ist, sollte das offen gesagt werden. Wer so arbeitet, baut Vertrauen auf und reduziert den Eindruck, der Text sei maschinell erzeugt.</p><p>Der Blick nach vorn ist pragmatisch. Die Modelle werden besser im Umgang mit längeren Kontexten. Werkzeuge zum Abruf interner Daten werden schneller und genauer. Multimodale Fähigkeiten gewinnen an Gewicht, also das Zusammenspiel von Text, Bild, Audio und Video. Für den Alltag bedeutet das weniger Klicks, mehr Automatisierung bei Routineaufgaben und mehr Zeit für die Teile der Arbeit, in denen Urteilskraft und Kreativität gefragt sind.</p>								</div>
