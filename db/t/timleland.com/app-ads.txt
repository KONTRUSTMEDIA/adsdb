				<p>After years of reliable camera streaming, I recently upgraded my <a href="https://timleland.com/wyze-surveillance-monitor-using-raspberry-pi/">RTSP Camera Dashboard</a> from a <a href="https://amzn.to/43Bg3M6">Raspberry Pi</a> to a <a href="https://amzn.to/4iCWGGJ">Dell Mini OptiPlex</a> with Ubuntu. I am still using the <a href="https://amzn.to/4hgNc2N">Wyze Cameras</a> flashed with the RTSP firmware, which <span style="box-sizing: border-box; margin: 0px; padding: 0px;">works great as long as your <a href="https://amzn.to/4iidX8p" target="_blank" rel="noopener">Wi-Fi connection</a> is reliable. I also rewrote the camera streaming software using C++.  This setup lets you live stream four cameras to a monitor, which works great for a DIY home security system.</span></p>
<h3>Setup</h3>
<div>
<ol>
<li>Copy the <a href="https://gist.githubusercontent.com/timleland/4c55cbf55b3d98c019177a4ea36860a1/raw/0ec257ac509fb93b7cce10b185e7980c373bb022/gistfile1.txt">C++ code</a> and create a new file called rtsp_viewer.cpp</li>
<li>Update the 4 RTSP URLs in the rtsp_viewer.cpp file  to be your camera URLS</li>
<li>sudo apt install libopencv-dev libsdl2-dev</li>
<li>g++ rtsp_viewer.cpp -o rtsp_viewer `pkg-config &#8211;cflags &#8211;libs opencv4 sdl2` -pthread</li>
<li>./rtsp_viewer</li>
<li>Setup this application to start on reboot</li>
</ol>
